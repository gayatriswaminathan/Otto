How Otto is built

Otto is an autonomous TPM: it reads where a program lives, keeps a live model of it, and ships an honest weekly status with a readiness scorecard and a real dependency map. Here's the architecture, the $0-stack mapping, and the tech behind it.

System architecture

Data flows top → bottom. Secrets stay server-side; connectors are read-only; every number traces to source.

Mapped to the $0 AI Architecture Stack

Each green layer is a tier from the reference $0 stack. The text is what Otto actually uses.

Tech stack

JavaScript end-to-end · zero runtime dependencies · free / low-cost tiers.

The idea that holds it together: everything funnels through one canonical program-state model, so any source plugs in without touching the engine. The engine does all the judgment with no LLM (cheap, testable, honest); Claude only writes the prose. D1 gives Otto memory for week-over-week deltas and cadence.
